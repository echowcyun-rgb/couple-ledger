# 房号制改造 - Cursor 执行命令

将情侣账本从单人使用改为多人隔离的「房号制」模式。
按顺序执行以下 4 个步骤。

---

## 步骤 0：前置准备

> **在浏览器打开 Supabase 面板执行 SQL。** 不经过 Cursor。

1. 打开 https://supabase.com/dashboard → 点你的项目
2. SQL Editor → 复制粘贴下面两段 SQL **分别执行**

**先执行：清空旧数据**
```sql
DROP TABLE IF EXISTS transactions CASCADE;
DROP TABLE IF EXISTS goals CASCADE;
DROP TABLE IF EXISTS members CASCADE;
DROP TABLE IF EXISTS import_batches CASCADE;
DROP TABLE IF EXISTS couples CASCADE;
```

**再执行：建新表（含 room_id）**
```sql
-- 1. 情侣房间表
CREATE TABLE IF NOT EXISTS couples (
  room_id TEXT PRIMARY KEY,
  created_at BIGINT NOT NULL DEFAULT (EXTRACT(EPOCH FROM NOW()) * 1000)
);

-- 2. 交易记录表（带 room_id）
CREATE TABLE IF NOT EXISTS transactions (
  id TEXT PRIMARY KEY,
  room_id TEXT NOT NULL REFERENCES couples(room_id),
  date TEXT NOT NULL,
  type TEXT NOT NULL CHECK (type IN ('out', 'in', 'save')),
  amount REAL NOT NULL,
  category_key TEXT NOT NULL,
  member_id TEXT NOT NULL,
  note TEXT DEFAULT '',
  status TEXT DEFAULT 'confirmed',
  recorder TEXT,
  created_at BIGINT NOT NULL,
  updated_at BIGINT NOT NULL DEFAULT (EXTRACT(EPOCH FROM NOW()) * 1000)
);
CREATE INDEX IF NOT EXISTS idx_transactions_room_date ON transactions(room_id, date DESC);
CREATE INDEX IF NOT EXISTS idx_transactions_room_type ON transactions(room_id, type);

-- 3. 存钱目标表（带 room_id）
CREATE TABLE IF NOT EXISTS goals (
  id BIGINT PRIMARY KEY,
  room_id TEXT NOT NULL REFERENCES couples(room_id),
  name TEXT NOT NULL,
  emoji TEXT DEFAULT '★',
  current REAL DEFAULT 0,
  target REAL NOT NULL,
  contributions JSONB DEFAULT '{}',
  history JSONB DEFAULT '[]',
  updated_at BIGINT NOT NULL DEFAULT (EXTRACT(EPOCH FROM NOW()) * 1000)
);
CREATE INDEX IF NOT EXISTS idx_goals_room ON goals(room_id);

-- 4. 成员表（带 room_id）
CREATE TABLE IF NOT EXISTS members (
  id TEXT PRIMARY KEY,
  room_id TEXT NOT NULL REFERENCES couples(room_id),
  name TEXT NOT NULL,
  avatar TEXT DEFAULT '',
  gender TEXT DEFAULT 'other',
  payday INTEGER DEFAULT 10,
  updated_at BIGINT NOT NULL DEFAULT (EXTRACT(EPOCH FROM NOW()) * 1000)
);
CREATE INDEX IF NOT EXISTS idx_members_room ON members(room_id);

-- 5. 导入批次表（带 room_id）
CREATE TABLE IF NOT EXISTS import_batches (
  id BIGINT GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
  room_id TEXT NOT NULL REFERENCES couples(room_id),
  ids JSONB NOT NULL,
  source TEXT NOT NULL,
  recorder TEXT NOT NULL,
  count INTEGER NOT NULL,
  time TEXT NOT NULL,
  created_at BIGINT NOT NULL DEFAULT (EXTRACT(EPOCH FROM NOW()) * 1000)
);
CREATE INDEX IF NOT EXISTS idx_import_batches_room ON import_batches(room_id);
```

```typescript

### 1.3 lib/supabase.ts
所有数据库查询操作增加 `.eq('room_id', roomId)` 条件。

具体改法：
- 在每个 `supabase.from(...).select(...)` 后面加 `.eq('room_id', roomId)`
- 在每个 `supabase.from(...).delete()` 后面加 `.eq('room_id', roomId)`
- 在每个 `supabase.from(...).upsert()` 或 `.insert()` 的数据对象中加 `room_id: roomId`
- 在 `pushToCloud` 函数中，每条推送到云的数据都带上 `room_id`

提示：各函数签名需要增加 `roomId: string` 参数。
例如 `pullTransactions(roomId: string)`、`pushTransaction(tx: Transaction, roomId: string)` 等。

### 1.4 lib/storage.ts

- 所有调 supabase 的地方把 roomId 传进去

## 步骤 3：接入页面

```
### 修改 app/page.tsx

在 `export default function Page()` 中：
1. 导入 RoomSetup 组件：`import { RoomSetup } from "@/components/modals/RoomSetup"`
2. 在 hydration 完成后，检查 localStorage 是否有 roomId
3. 如果没有 roomId（首次使用），显示 RoomSetup 弹窗
4. 如果有 roomId，正常显示主界面（现有代码不变）

示例逻辑：
```tsx
const [showRoomSetup, setShowRoomSetup] = useState(false)

// 在 hydration 的 useEffect 里
useEffect(() => {
  if (ledger.hydrated) {
    const savedRoom = localStorage.getItem('couple-room-id')
    if (!savedRoom) {
      setShowRoomSetup(true)
    }
  }
}, [ledger.hydrated])

// 渲染
{showRoomSetup ? <RoomSetup onDone={(roomId) => {
  localStorage.setItem('couple-room-id', roomId)
  setShowRoomSetup(false)
  // 重新加载数据
  window.location.reload()
}} /> : (
  // ... 现有主界面代码
)}
```
```
